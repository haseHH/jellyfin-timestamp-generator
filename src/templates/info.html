Custom formatted Playback Timestamp

To get a preformatted string with your chosen format, call the /session/{username} endpoint. Optionally, add these query parameters:

  • format: A string into which the timestamps should be inserted into. Some special characters might need to be URL-encoded. These tokens can be used and will be replaced:
    • %p: play_state will indicate whether the player is paused, either '▶' or '⏸'.
    • %c: current is the timestamp of where the progress currently is.
    • %t: total is the total length of the currently playing media.
    • %r: remaining is the remaining play time.
    • %s: A non-breakable space. Usable instead of the URL-encoded %20 which cannot be doubled up.
    The default format is %p%s%s%c%s/%s%t%s%s-%r and results in something like this:
    ▶ 00:04 / 43:12 -43:08
  • font: Choose any font installed on your system.
  • font_size: CSS compatible size, e.g. 20px, defaults to a font size of 40 pixels.
  • text_color: CSS compatible size, e.g. #0000FF, defaults to black.
  • outline_color: CSS compatible size, e.g. #00FF00, defaults to white.
  • outline_width: CSS compatible size, e.g. 20px, defaults to 0 pixels/being hidden.

If you're using this within OBS, remove any custom CSS in the browser source to prevent flickering of the text and use the styling parameters.

An example call with all options in use could be:

/session/hase?font=Courier%20Prime&font_size=40px&text_color=white&outline_color=black&outline_width=1px&force_hour=true&format=%p%s%c
▶ 0:44:34

API for your own purposes

If you want to use the parsed timestamps in another application, call the /api/session/{username} endpoint to get a JSON-formatted response:

{"play_state":"▶","current":"00:04","total":"43:12","remaining":"43:08"}
